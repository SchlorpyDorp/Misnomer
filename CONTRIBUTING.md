# Contributing to Misnomer

Thank you for your interest in contributing to Misnomer.

## Reporting Bugs

If you encounter a bug, please open a GitHub Issue and include:

- Device model
- Android version
- ROM
- Root solution (KernelSU, KernelSU Next, Magisk, APatch, etc.)
- Misnomer version
- Steps to reproduce the issue

Screenshots and logs are always appreciated.

## Suggesting Features

Feature requests are welcome.

Before opening a new request, please check the Discussions page to see if your idea has already been suggested.

## Pull Requests

If you'd like to contribute code:

1. Fork the repository.
2. Create a new branch.
3. Make your changes.
4. Test your changes.
5. Open a Pull Request with a clear description of what you've changed.

Please keep changes focused and avoid unrelated formatting changes.

## Code Style

- Keep the code simple and readable.
- Preserve the existing project structure where possible.
- Test changes before submitting.

Thank you for helping improve Misnomer.
